  • static 默认就是static文档流,static 定位 不存在谁是谁的定位上下文，他们共享相同的起点。
  • relative 相对它原来在文档中的位置。原来的空间依然占据着
  • absolute 对应定位上下文body(可设定其他定位上下文)，会跟随页面滚动。原来的空间会被清空.
  • fixed 对应定位上下文window，区别于absolute 不会跟随页面滚动。原来的空间会被清空. （很少用）
  • 只要将父元素设为relative,父元素就会成为子元素的定位上下文。
